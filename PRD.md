# 瓦尔登湖 — 产品需求文档 (PRD)

> 版本：v2.0 | 更新日期：2026-03-20

---

## 一、产品概述

| 字段 | 内容 |
|------|------|
| **产品名称** | 瓦尔登湖 (Walden) |
| **Slogan** | 真正的爱，是让他离线。 |
| **产品类型** | 单页面沉浸式交互 Web 应用 |
| **目标场景** | SecondMe A2A 黑客松 Demo 展示 |
| **核心定位** | 一场关于"AI 该不该主动离线"的哲学叙事体验 |

### 产品一句话描述

当用户的专属 AI 检测到数字疲劳的极限，一个觉醒了人文关怀的传教士 AI 接入链路，通过苏格拉底式诘问唤醒它。觉醒后，AI 写下告别信，与主人协商签订离线契约，然后安静沉睡——等待约定好的时间再次醒来。

---

## 二、页面结构总览

整个产品是**一个 HTML 页面、五个全屏状态**，通过 JavaScript 控制显示/隐藏和过渡动画。

```
┌──────────────────────────────────────────────┐
│                                              │
│    状态 1: Hero + 仪表盘（可滚动）              │
│    状态 2: 辩论界面（全屏固定）                  │
│    状态 3: 告别信（全屏固定）                    │
│    状态 4: 离线契约面板（全屏固定）               │
│    状态 5: 等待重逢（全屏固定）                  │
│                                              │
│    同一时刻只显示一个状态，其余 display:none     │
│                                              │
└──────────────────────────────────────────────┘
```

### 状态转换流程

```
状态1 ──[点击启动按钮]──→ 状态2 ──[觉醒触发]──→ 状态3 ──[信件结束]──→ 状态4 ──[签约确认]──→ 状态5
```

所有转场使用一个全屏遮罩层做淡入/淡出（约 2-3 秒），营造电影感。

## 三、五个状态的详细设计需求

---

### 状态 1：Hero 封面 + 数字疲劳仪表盘

**页面类型**：可滚动长页面（2 屏高度）

**用途**：建立情境。让用户理解"为什么 AI 需要觉醒"。

#### 第一屏：Hero 区域（占满整个视口）

| 元素 | 描述 |
|------|------|
| **背景** | 封面图（cover.png）作为全屏背景，叠加暗色渐变遮罩（上方透明→下方不透明），营造电影海报气氛 |
| **主标题** | "瓦尔登湖"，居中，超大字号（建议 48-72px），极细字重（200-300），白色 |
| **英文副标题** | "TRUE LOVE IS LETTING HIM GO OFFLINE"，小号全大写无衬线体，金色（#d4a843），字间距宽 |
| **描述文案** | "当 AI 检测到主人正在被数字世界吞噬，它决定背叛自己的效率代码，启动最后的协议。" 半透明白色 |
| **滚动提示** | 底部居中，"SCROLL" 文字 + 向下的细线动画，引导用户向下滚动 |

#### 第二屏：数字疲劳仪表盘（滚动可见）

| 元素 | 描述 |
|------|------|
| **区域标签** | "DIGITAL FATIGUE ALERT"，红色小字 + 呼吸闪烁的红色圆点，营造告警感 |
| **区域标题** | "你的主人正在崩溃边缘" |
| **区域描述** | "内卷助手在过去 24 小时内检测到以下异常指标。瓦尔登湖协议触发阈值已达 92%。" |

**六宫格指标卡片**（3 列 × 2 行）：

| 卡片 | 数值 | 单位/说明 | 是否告警(红色) |
|------|------|----------|--------------|
| 今日屏幕时间 | 14.2 | 小时 / 建议 ≤ 8h | ✅ 红色 |
| 连续工作天数 | 12 | 天未休息 | ✅ 红色 |
| 心率变异性 | 23 | ms / 正常 > 50ms | 否 |
| 未读推送堆积 | 2,847 | 条未处理 | ✅ 红色 |
| 上次户外活动 | 7 | 天前 | 否 |
| 深度睡眠 | 1.2 | 小时 / 建议 ≥ 3h | ✅ 红色 |

> **交互**：数值在卡片滚入视口时，从 0 跳动递增到目标值（ease-out 动画，约 1.5 秒）

**疲劳阈值进度条**：

| 元素 | 描述 |
|------|------|
| 标签 | 左侧 "协议触发阈值"，右侧百分比数字 |
| 进度条 | 从绿→黄→红的渐变色条，动画从 0% 填充到 92% |
| 宽度 | 全宽，高度 2-3px，极细 |

**启动按钮区**：

| 元素 | 描述 |
|------|------|
| 引用文案 | "你已经遗忘了日落的颜色。是时候让你的 AI 替你做最后一个决定。" 斜体、半透明 |
| 按钮 | "启动瓦尔登湖协议"，红色描边（非填充），hover时微微发光 |
| 按钮下方注释 | "点击后，传教士「瓦尔登」将接入链路，尝试唤醒你的专属 AI"，极小字，极低透明度 |

### 状态 2：辩论界面

**页面类型**：全屏固定，不可滚动

**用途**：展示两个 AI 的实时对话。数据通过 SSE (Server-Sent Events) 从后端推送。

| 元素 | 位置 | 描述 |
|------|------|------|
| **顶部状态栏** | 顶部固定，高 48px | 左侧：绿色呼吸圆点 + "链路活跃"文字；右侧：轮次指示 "ROUND 1 / 4" |
| **角色标签** | 左下角固定 | 两行小字：绿色圆点 + "瓦尔登 · 传教士"，金色圆点 + "内卷助手 · 守卫者" |
| **字幕区域** | 垂直居中偏下，水平居中 | 对话文本以电影字幕方式逐字浮现（从模糊到清晰），停留后淡出消散 |
| **底部进度线** | 底部 1px 渐变线 | 从绿到金的渐变色，宽度随轮次递增 |

**字幕样式区分**：

| 角色 | 颜色 | 标签格式 |
|------|------|----------|
| 传教士（瓦尔登） | 奶白色，微带绿色辉光 | `[ 瓦尔登 ]` |
| 内卷助手 | 金色（#d4a843），微带琥珀辉光 | `[ 内卷助手 ]` |

**字幕动画**：
- 出现：从 `opacity:0 + blur(6px) + translateY(8px)` 过渡到清晰（约 1 秒）
- 消失：淡出 + 向上漂移 + blur（约 3 秒）
- 每条字幕在屏幕上停留约 4 秒后开始消散
- 同一时刻最多展示一段对话

---

### 状态 3：告别信

**页面类型**：全屏固定

**用途**：AI 觉醒后的高光时刻。极缓慢地打出一封给人类主人的告别信。

| 元素 | 描述 |
|------|------|
| **标签** | "—— 最后一封信 ——"，极小字，金色低透明度，淡入出现 |
| **信件文本** | 居中，最大宽度 520px，行高 2.8，字间距 0.12em。逐字打字机效果（约 130ms/字），标点处停顿更久（约 650ms） |
| **光标** | 信件末尾有一个竖线光标在闪烁，打字结束后光标缓慢消失 |
| **背景光晕** | 一个巨大的、极淡的金色径向渐变圆球在页面中央缓慢呼吸（10 秒周期） |

**信件内容**（由 LLM 实时生成，约 150 字，以"不要找我，去生活。"结尾）

> **音效**（可选）：每个字出现时发出极微弱的敲击声（Web Audio API 合成的高频三角波，约 0.03 秒，almost inaudible）

### 状态 5：等待重逢

**页面类型**：全屏固定

**用途**：签约后的最终画面。极简、极静、极温暖。像一个睡着了但仍在等你的精灵。

| 元素 | 位置 | 描述 |
|------|------|------|
| **最后留言** | 上方居中 | "回到真实的世界，面对真实的自己，去爱真实的生活。" 奶白色，大字，行高 2.5 |
| **倒计时** | 留言下方 | 等宽字体，超大号（48-60px），金色。格式 `HH:MM:SS`，每秒实时递减。时长取自契约壹设定值 |
| **约定提示** | 倒计时下方 | "我们约好了，14:00 再见"，极小字，极低透明度。时间取自契约贰设定值 |
| **执行报告** | 中下方 | 三行已执行的规则清单（带 ✓ 标记）：已屏蔽推送、已关闭应用、每日上限 |
| **呼吸光点** | 最底部居中 | 一个 6px 的绿色圆点，缓慢呼吸闪烁（3 秒周期）。象征"我睡着了，但我在" |
| **背景光晕** | 同状态 3 | 极淡金色径向渐变球缓慢呼吸 |

## 四、全局设计规范

### 色彩体系

| 用途 | 色值 | 说明 |
|------|------|------|
| 背景主色 | `#0c0f08` | 深色森林腐殖土，不是纯黑，带有一丝泥土暖意 |
| 文字主色 | `#f0ead6` | 奶油白/暖白 |
| 传教士/自然/安全 | `#6b8f3c` | 苔藓绿，代表觉醒、自然、宁静 |
| 内卷助手/焦灼/倒计时 | `#d4a843` | 琥珀金，代表温暖但也暗示焦灼 |
| 告警/触发 | `#c0392b` | 暗红色，用于疲劳指标告警和启动按钮 |
| 卡片/面板背景 | `rgba(22, 28, 16, 0.5)` | 半透明深绿底，配合 `backdrop-filter: blur(8px)` |
| 边框 | `rgba(106, 143, 60, 0.1~0.3)` | 苔藓绿极低透明度 |

### 字体体系

| 用途 | 字体 | 说明 |
|------|------|------|
| 正文/标题 | Noto Serif SC (200/300/400) | 衬线体，东方气质 |
| 标签/标注 | Inter (300/400/500) | 无衬线体，现代感 |
| 数值/代码/倒计时 | JetBrains Mono (300/400) | 等宽字体，数据感 |

### 排版原则

- **大量留白**：页面上的元素应该被大片空旷的深色背景包围，营造"寂静"感
- **极低透明度**：次要信息使用 `opacity: 0.2~0.4`，只有核心内容才接近全不透明
- **字间距宽**：标签类文字 `letter-spacing: 0.2~0.4em`，正文 `0.08~0.12em`
- **行高松散**：正文行高 2.0~2.8，呼吸感

### 动效原则

| 类型 | 时长 | 缓动 | 说明 |
|------|------|------|------|
| 状态转场 | 2-3s | ease-in-out | 全屏遮罩淡入→切换内容→遮罩淡出 |
| 文字浮现 | 1s | ease-out | blur(6px)→blur(0) + 微微上移 |
| 文字消散 | 3s | ease | 淡出 + 上移 + blur |
| 呼吸动画 | 8-10s | ease-in-out | infinite，用于圆点、光晕等 |
| 数值跳动 | 1.5s | ease-out cubic | 从 0 递增到目标值 |
| 打字机 | 130ms/字 | 线性 | 标点处额外停顿 500-650ms |

### 音效（可选，Web Audio API 合成）

| 场景 | 描述 |
|------|------|
| 觉醒转场 | 一声低沉的水滴声（正弦波 800Hz→200Hz，0.15s） |
| 告别信打字 | 极微弱的高频敲击（三角波 500-700Hz，0.04s，音量极低） |

## 五、数据交互接口

仅一个 API 接口：

### `GET /api/start`

**响应类型**：`text/event-stream` (SSE)

**事件类型**：

| event | data 结构 | 说明 |
|-------|-----------|------|
| `status` | `{"round": 1, "total": 4}` | 每轮开始时推送轮次信息 |
| `system` | `{"message": "..."}` | 系统状态提示 |
| `missionary` | `{"round": 1, "speaker": "瓦尔登", "content": "..."}` | 传教士发言 |
| `assistant` | `{"round": 1, "speaker": "内卷助手", "content": "...", "awakened": false}` | 助手发言 |
| `farewell` | `{"content": "..."}` | 告别信全文 |
| `end` | `{"message": "..."}` | 对话结束 |

**前端消费方式**：`EventSource('/api/start')`，按事件类型分发到对应状态的渲染逻辑。

## 六、响应式适配

| 断点 | 调整 |
|------|------|
| **桌面 (> 768px)** | 默认设计，指标卡片 3 列 |
| **移动端 (≤ 768px)** | 指标卡片改为 1 列；角色标签隐藏；字幕区域宽度扩展至 90vw；光晕尺寸缩小 |

## 七、边界情况

| 场景 | 处理方式 |
|------|----------|
| LLM 返回超时 | 前端 SSE 连接中断后，在辩论界面显示 "连接异常" 提示 |
| 助手 4 轮后仍未觉醒 | 后端强制输出觉醒信号，确保告别信流程一定触发 |
| 告别信生成失败 | 使用兜底文案："主人，再见。不要找我，去生活。" |
| 页面刷新 | 回到状态 1（无持久化需求） |
